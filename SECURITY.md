# Security Policy

This is a research framework for behavioral digital attribution. The framework is released as open-source academic research.

## Reporting Vulnerabilities

If you discover a security vulnerability in the framework implementation, please report it through GitHub security advisories.

Due to the research nature of this framework, security considerations are mainly relevant to:
- Prototype implementation security
- Data handling in experiments
- API design patterns

## Responsible Use

This framework is released for research purposes only. Behavioral attribution technology has dual-use potential. Users are responsible for ensuring compliance with local laws and ethical guidelines.

## Limitations

The framework currently lacks:
- Input validation hardening
- Security audit for production use
- Privacy compliance assessment

This is intentional as the framework is presented as research methodology, not operational software.
